01 — Introduction

Mon parcours &
mes ambitions

Mon entrée dans l'informatique n'est pas un accident de parcours — c'est une décision que j'ai prise après mon BTS SIO option SLAM à Massy, quand j'ai compris que je voulais aller plus loin que le développement applicatif classique. Intégrer l'IUT de Villetaneuse en DUT Passerelle Sciences des données et Informatique, puis enchaîner sur le BUT 3 parcours Réalisations d'Applications, m'a permis de consolider une base technique solide tout en restant ancré sur des projets concrets et des mises en situation réelles.

Ce qui me caractérise, c'est la capacité à m'adapter vite à des environnements variés : de l'IT support sous pression lors des Jeux Olympiques Paris 2024 chez ATOS, au développement d'une API REST sécurisée chez VÆG Conformité, en passant par un stage web à VINCI AART dès ma première année de BTS. Chaque expérience a ajouté une couche à ma compréhension du développement logiciel en conditions réelles.

Je vise une poursuite en école d'ingénieur avec une orientation cybersécurité ou analyste. Ce portfolio est le bilan de mes trois ans de BUT — ce que j'ai appris, ce que j'aurais fait différemment, et ce que j'apporte à une équipe technique.

02 — Référentiel BUT

Les six compétences

Compétence 01
Réaliser
Développer des applications informatiques simples et avancées

C'est la compétence qui résume le mieux mon identité de développeur. Du front-end React au back-end FastAPI, en passant par le C# WinForms et le mobile Flutter, j'ai touché à des paradigmes variés — et c'est volontaire. Mon stage chez VÆG Conformité a été la vraie mise en pratique : concevoir un espace administrateur RGPD complet avec une API REST sécurisée par JWT, une base MySQL journalisée, en architecture MVC avec sprints Agile. Ce n'est pas un exercice de cours — c'est une livraison réelle. Mon projet BookWise (Flutter/Dart/SQLite) et l'application de gestion de magasin en C#/.NET témoignent de ma capacité à couvrir mobile et desktop. Le projet THALES m'a exposé à l'IA appliquée via les LLMs avec l'API Groq.

✅ Points forts
  • Full-stack : React, Vue.js, Angular côté front ; FastAPI, Node.js, PHP côté back
  • Architecture MVC, API REST sécurisée JWT, principes Clean Code
  • Mobile Flutter/Dart et desktop C#/.NET
⚡ Points de vigilance
  • Anticiper la phase de tests dès le départ — ils ont souvent été ajoutés en fin de sprint
  • Consolider SpringBoot pour les architectures Java orientées production
🎯 Traces principales
  • Stage VÆG Conformité : API REST JWT + tableau de bord RGPD, MySQL, React, MVC
  • BookWise (SAÉ BUT 3) : App mobile recommandation livres, Flutter/Dart/SQLite, équipe de 7
  • App gestion magasin : C#/.NET WinForms, POO, base MySQL
  • Projet THALES (BUT 2) : Scripts Python FTP + API Groq + Streamlit

Compétence 02
Optimiser
Proposer des applications informatiques optimisées selon des critères précis

L'optimisation, ce n'est pas juste rendre le code plus rapide — c'est le rendre plus juste. Dans BookWise, j'ai conçu un moteur de recommandation adaptatif 100% local : pas de serveur externe, pas d'API tierce, une logique basée sur les favoris, l'historique et l'analyse de mots-clés. Ce choix d'architecture est une décision d'optimisation : réduire les dépendances, garantir la réactivité, protéger les données. Dans le projet THALES, le scoring de pertinence des articles académiques via LLM nous a obligés à penser l'efficience des appels API pour rester dans les limites de coût et de latence. Sur VÆG, j'ai optimisé les requêtes MySQL avec journalisation pour ne pas dégrader les performances malgré le tracking comportemental.

✅ Points forts
  • Conception d'algorithmes locaux sans dépendances externes superflues
  • Optimisation des requêtes SQL (indexation, réduction des SELECT *)
  • Approche coût/performance dans les intégrations LLM
⚡ Points de vigilance
  • Aller plus loin dans l'analyse formelle de complexité algorithmique
  • Systématiser le benchmarking des solutions avant de choisir une approche
🎯 Traces principales
  • BookWise : Moteur de recommandation adaptatif local (favoris + historique + mots-clés)
  • Projet THALES : Scoring LLM des publications 6G avec gestion de l'usage API
  • VÆG Conformité : Optimisation des requêtes MySQL avec journalisation et tracking
AlgorithmiqueOptimisation SQLLLM scoringArchitecture locale
Compétence 03
Administrer
Déployer des services dans une architecture réseau

L'administration système, je l'ai approchée par deux angles très différents. En cours de BUT, via Docker, les pipelines CI/CD et Jenkins. En entreprise, de façon concrète lors de mon stage chez ATOS aux Invalides pendant les JO Paris 2024 : gérer des tickets d'incidents en temps réel sur un site olympique critique, installer et reconfigurer des équipements sous pression, respecter des procédures ITSM strictes via Jira. En développement, j'utilise Git/GitHub comme outil de travail quotidien avec des workflows de branches et des pratiques qui s'inscrivent dans une logique DevOps. La conteneurisation Docker fait partie de ma stack pour isoler les environnements.

✅ Points forts
  • Gestion d'incidents IT en environnement critique (JO Paris 2024)
  • Outils ITSM (Jira), respect des procédures sous pression
  • Conteneurisation Docker, Git avancé et workflows de branches
⚡ Points de vigilance
  • Approfondir Docker Compose et les pipelines CI/CD Jenkins en autonomie
  • Monter en compétence sur l'administration Linux en production
🎯 Traces principales
  • ATOS — JO Paris 2024 : Support IT N1, gestion d'incidents Jira, install/config équipements sur site olympique
  • Stage VÆG : Gestion de l'environnement de dev et déploiement de l'application web
  • Cours BUT : Docker, Git avancé, CI/CD, Jenkins
DockerGitJiraCI/CDJenkinsITSM
Compétence 04
Gérer
Concevoir et gérer des données de l'entreprise

Les données, je les manipule depuis le BTS — mais c'est le BUT qui m'a appris à les concevoir proprement. Chez VÆG Conformité, j'ai modélisé de A à Z une base MySQL avec journalisation des actions et tracking comportemental des utilisateurs : ça demande de penser la donnée comme un outil d'audit, pas juste un stockage. Dans BookWise, j'ai conçu le schéma SQLite du moteur de recommandation local. Le projet THALES m'a familiarisé avec le traitement de données semi-structurées (CSV, FTP) et leur exploitation via des LLMs. J'ai aussi travaillé avec MongoDB et Redis dans le cadre des cours de NoSQL.

✅ Points forts
  • Modélisation relationnelle (MySQL, PostgreSQL, SQLite) avec traçabilité
  • Traitement de données semi-structurées (CSV, FTP) en Python/Pandas
  • Initiation aux bases NoSQL (MongoDB, Redis)
⚡ Points de vigilance
  • Documenter systématiquement le modèle de données dès la conception
  • Approfondir l'optimisation sur gros volumes
🎯 Traces principales
  • VÆG Conformité : Base MySQL avec journalisation + tracking, API REST pour le tableau de bord
  • BookWise : Schéma SQLite pour moteur de recommandation local
  • Projet THALES : Extraction FTP, export CSV, visualisation Streamlit + LLM
MySQLPostgreSQLSQLiteMongoDBRedisPandas
Compétence 05
Conduire
Satisfaire les besoins des utilisateurs au travers d'une approche projet

Conduire un projet, ça commence par comprendre ce que l'utilisateur veut vraiment — et pas seulement ce qu'il dit. Chez VÆG Conformité, les sprints Agile m'ont appris à livrer de la valeur de façon itérative, à retravailler les specs en cours de route sans repartir de zéro. Sur BookWise (7 personnes), on a rédigé un vrai cahier des charges, géré des jalons, et livré une application fonctionnelle. Les cours de droit du numérique, RGPD et méthodes de gestion de projet du BUT 3 m'ont donné le cadre légal et méthodologique pour contextualiser mes choix techniques. La documentation — rédiger des procédures claires — est aussi quelque chose que j'ai pratiqué chez ATOS comme dans mes stages de développement.

✅ Points forts
  • Méthode Agile/Scrum appliquée en entreprise (sprints, livrables définis)
  • Gestion de projet en équipe avec jalons et documentation (7 personnes)
  • Connaissance du cadre RGPD et application concrète chez VÆG
⚡ Points de vigilance
  • Formaliser les rétrospectives de sprint pour capitaliser sur les apprentissages
  • Maintenir la documentation technique à jour en cours de développement
🎯 Traces principales
  • VÆG Conformité : Sprints Agile, livraison d'un espace admin conforme RGPD
  • ATOS JO 2024 : Rédaction de procédures, documentation technique, protocoles ITSM
  • BookWise SAÉ : Cahier des charges, gestion de projet Git/GitHub Project, équipe de 7
Agile / ScrumRGPDTDDClean CodeDocumentationJira
Compétence 06
Collaborer
Travailler dans une équipe informatique

Le travail en équipe, c'est souvent là où les choses se jouent vraiment. Sur BookWise (7 personnes), on a géré conflits Git, désaccords de conception et délais — en restant soudés. Le projet THALES nous a mis en contact avec un vrai client externe, ce qui change les exigences et la communication. Chez ATOS, l'équipe IT travaillait en rotation sur un site olympique : la passation d'information entre collègues était critique. J'ai aussi collaboré directement en binôme avec mon camarade de promo Razi Hadj Amor sur le web scraping 6G — une expérience de pair programming intensif sur un projet de bout en bout. La collaboration, pour moi, c'est aussi savoir quand demander de l'aide et quand en donner.

✅ Points forts
  • Git collaboratif maîtrisé : branches, pull requests, résolution de conflits
  • Communication dans des contextes variés (entreprise, client externe, équipe JO)
  • Pair programming et revue de code entre pairs
⚡ Points de vigilance
  • Formaliser les conventions de branches et commits dès le démarrage d'un projet
  • Améliorer la passation documentaire en fin de sprint
🎯 Traces principales
  • BookWise : Collaboration Git/GitHub Project en équipe de 7, gestion de conflits, revues collectives
  • Projet THALES : Travail avec un interlocuteur professionnel externe, livrables définis
  • ATOS JO 2024 : Travail en équipe sous pression, passation d'information critique
  • Web Scraping 6G : Binôme de pair programming sur un projet technique complet

06 — Synthèse finale

Bilan de compétences &
perspectives d'avenir

« Ce qui distingue un développeur d'un bon développeur, ce n'est pas le nombre de langages qu'il connaît — c'est sa capacité à livrer quelque chose qui fonctionne, dans un contexte contraint, en équipe. Le BUT m'a appris les deux. »

Arriver au terme de ce BUT Informatique, c'est mesurer le chemin parcouru depuis le BTS SIO : de l'apprentissage des fondamentaux du développement à la livraison d'applications réelles en entreprise, en passant par des projets avec des acteurs comme THALES et une expérience hors-norme aux Jeux Olympiques de Paris 2024. Ce parcours n'a pas été linéaire — il a été fait de choix, d'adaptations et parfois de remises en question.

Ma force principale réside dans la polyvalence technique doublée d'une vraie capacité à travailler dans des environnements professionnels contraints et variés. J'ai développé en C#, en Python, en JavaScript et en Dart. J'ai livré en Agile, géré des bases de données en conditions réelles, et répondu à des incidents IT sous pression olympique. Cette diversité n'est pas de la superficialité — c'est une base qui me permet de m'adapter vite à n'importe quel contexte technique.

Les axes d'amélioration que j'identifie — les tests écrits trop tard, la documentation parfois négligée pendant les sprints — sont précisément ceux sur lesquels je veux travailler dans la suite de mon parcours. Une école d'ingénieur, c'est l'endroit pour consolider la rigueur que le rythme de formation ne permet pas toujours d'atteindre pleinement.

  • Polyvalence full-stack (front, back, mobile, desktop)
  • Expériences professionnelles variées en conditions réelles
  • Capacité d'adaptation rapide à de nouveaux environnements

  • Travail en équipe et gestion Git collaborative
Axes de progression

  • Approfondir la cybersécurité et les architectures sécurisées
  • Progresser sur Docker Compose et les environnements conteneurisés
